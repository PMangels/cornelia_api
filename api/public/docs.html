Quick start guide

To get information from the API, an HTTP GET request should be sent to . This section will explain how to form these requests based on the documentation on this page. In the next section, a few coding examples are given for some concrete scenarios.

A request for a function is made by appending the function to . For example to use the /archives/count function, the request should read /archives/count. The parameters are separated by a question mark from the function name and a list of parameters has an ampersand between each parameter. The name of the parameter must be typed and assigned with an equal sign. For example, to get the id of the actor whose first and surname are given as Antoon Sallaert, the request would look as follows: /actor/id?first_name=Antoon&surname=Sallaert

The requests return a json file. If the documentation specifies that a list will be returned, this will take the form of a list of object. Otherwise the json file will be one object with the fields that are specified in the return statements of the documentation.

An invalid request will result in an HTTP error. If an unrecognised method is requested, a 404 error will be returned to indicate the page was not found and if one of the parameters was not recognised, a 400 error will be returned to indicate a bad request. Any other conditions that will result in errors are indicated in the methods documentation.

Examples

In this section a few examples will be given to clarify the use of the API. These examples will be written in JavaScript, using the fetch function for submitting the HTTP request. If the code is not run in a browser, but instead in a node environment, this fetch function can be provided as follows:

const fetch = require("node-fetch")

This example shows how to access the information of an actor.

//Defining the http requests
const domain = "/";
const actor_info_url = "actor/info?id="

//Fetching the info of the actor
fetch(domain+actor_info_url+490).then(data=>{return data.json()}).then(res => {

//Logging the entire object to the console
console.log(res);

//Logging the gender of the actor
console.log(res.gender);

})

The code below will return a list of places where the parish contains the words "Onze-Lieve-Vrouw".

//Defining the http requests
const domain = "/";
const parish_search_url = "place/parish/search?name="

//Fetching the parishes whose name contain "Onze-Lieve-Vrouw".
fetch(domain+parish_search_url+"Onze-Lieve-Vrouw").then(data=>{return data.json()}).then(res => {

//Printing the places matching the search
console.log(res);

//We can loop through the list and only print those entries who match a condition,
//in this case that it's located in Brussels.
i = 0;
while (i < res.length){
if (res[i].city="Brussels"){console.log(res[i]);}
i+=1;
}

})

The output of one request can also be used as the input of the next request. This is demonstrated in the code below. It attempts to answer the following question: given the actor with id 490, find the place where their daughter Catharina was married.

//Defining the http requests
const domain = "/";
const offspring_url = "relationship/personal/offspring?id="
const life_personal_events_url = "actor/life_events/personal?id="

//Fetching all the offspring of the actor with id 490
fetch(domain+offspring_url+490).then(data=>{return data.json()}).then(res => {

//Searching for the child with the name Catharina
entry = {};
i = 0;
while (i < res.length){
if (res[i].first_name="Catharina"){entry = res[i];}
i+=1;
}

//Fetching the life events of Catharina
fetch(domain+life_personal_events_url+entry.id).then(data=>{return data.json()}).then(res => {

//Searching for Catharina's marriage
entry = {};
i = 0;
while (i < res.length){
if (res[i].role="married"){entry = res[i];}
i+=1;
}

//Logging the place of Catharina's wedding.
console.log(entry.place);

})})

/actor/id?first_name&surname

Returns the ids of the people matching the first and surname provided exactly.

params

first_name: The first name of the actor

surname: The surname of the actor

return

List of:

id: The id matching the given name

Table of contents

/actor/search?name&variants

Returns a list of actors matching the description given in name. This does not have to be the complete name and variants of this name that were recorded can be searched as well.

params

name: The name of the person that is being searched for.

variants (default=0): Whether or not to search the variants of names.

return

List of:

id: The id of the actor matching the description

first_name: The first name of the actor

surname: The surname of the actor

variant: The variant of the actor's name (if variants is 1)

variant_source_entry_id: The id of the entry containing the reference to the variant (if variants is 1)

error

400: if variants is not in (0,1)

Table of contents

/actor/info?id

Returns the information of the actor identified by the id.

params

id: The id of the actor

return

first_name: The first name of the actor

surname: The surname of the actor

gender: The gender of the actor (male or female)

incomplete: Indicates whether the record of the actor is incomplete (1=incomplete, 0=complete)

/actor/lifespan?id

Returns the lifespan of the person identified by the id, based on their baptism and their burial date.

params

id: The id of the actor

return

year_of_birth: The year in which the person was born, can be null

month_of_birth: The month in which the person was born, can be null

day_of_birth: The day in which the person was born, can be null

source_entry_birth_id: The id of the source entry which recorded this birth

year_of_death: The year in which the person died, can be null

month_of_death: The month in which the person died, can be null

day_of_death: The day in which the person died, can be null

source_entry_death_id: The id of the source entry which recorded this death
